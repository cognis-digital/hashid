When you find a scrambled password string — like the ones stored in a database dump or a leaked file — hashid tells you what kind of scrambling algorithm was used to create it. It also estimates how long it would realistically take for an attacker to crack that password, so you know how urgently you need to act. It works entirely from the command line with no setup, accepts one hash or a whole file full of them, and can output results as plain text or JSON for automated pipelines. Security analysts, developers auditing their own systems, and penetration testers use it to quickly triage password-storage weaknesses without needing to run any cracking tools themselves.
